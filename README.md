# 三线学习训练器

这是一个本地 Web 学习程序，面向三个目标。当前版本更偏“课文/长句驱动”，先用上下文帮助记忆，再用题目检查是否真的会用。

- 日语：初级阶段，围绕《大家的日语》前 15 课附近的词汇和语法做原创练习，不复制教材正文。
- 英语：以 IELTS 6.0 为起点，补土木工程和 AI 学术英语。
- 《逻辑哲学论》：从核心概念、概念关系和入门读法开始。

## 使用

直接用浏览器打开 `index.html`。进度会保存在浏览器的 `localStorage` 里。

在 iPhone 上使用时，建议用 Safari 打开后添加到主屏幕。页面已经适配安全区和底部操作栏，练习时主要点击底部的提交、忘了、模糊、会了。

你的通勤路线已经适配为四段：

- 站台/换乘：3 题左右，只做单手选择题，随时可停。
- 南与野 → 赤羽：8 题左右，适合读日语/英语课文长句，进入学习状态。
- 赤羽 → 王子：5 题左右，短段快练，不安排复杂打字。
- 王子 → 东大前：10 题左右，适合长句、默写、口语复述或哲学段落精读。

## 第一版功能

- 选择题、填空题、组句题、自评口语/解释题。
- 日语和英语增加“课文长句 / 长句精读”题组，题卡会显示上下文、要点和可选译文。
- 每天可以记录“今天学了什么、想用什么形式练、当前状态、卡住的点”。
- 按“忘了 / 模糊 / 会了”评分，并自动安排下次复习。
- 今日到期、智能推荐、弱项加练、混合挑战四种模式。
- 智能推荐会按“到期复习、今日内容、弱项、偏好形式、未见课文”几个篮子组队，逻辑更简单可解释。
- 学习档案会根据每日记录、正确率、未见题、到期题和错题标签推荐下一轮重点。
- 同一张题明显卡住时，程序会自动生成巩固题，逐步贴合学习进度。
- 每日记录会自动生成少量今日复述题、默写题和难点解释题。
- 英语和日语题目支持浏览器语音朗读。
- 可以在界面里添加自定义填空题，也可以粘贴老师课文、英文长句或哲学段落作为上下文。
- iPhone 移动端优化：科目、题组、模式改为横向滑动条，底部按钮固定在安全区上方，输入框字号避免 iOS 自动放大。
- 通勤模式：按“南与野→赤羽→王子→东大前”的实际路线分段推荐题型和题量。
- 数据同步：可以导出/复制/下载备份，并在另一台设备上导入合并。

## 推荐用法

1. 每天先填“今日记录”，写下老师讲的课文、长句、知识点和卡住的地方。
2. 选择当天想练的形式：课文长句、做题反馈、听力朗读、口语复述或默写输出。
3. 点“智能推荐”，程序会优先安排今日内容、到期错题和适合当天状态的题量。
4. 如果老师讲了新课文，把关键句粘贴到“自定义题”的“课文/长句”里。
5. 遇到答错题不要急着重背，按“忘了”或“模糊”，程序会把它排进后续复习。

## 放到 iPhone

电脑本地文件路径 `D:\codex\学习程序\index.html` 不能直接在 iPhone Safari 打开。推荐三种方式：

1. 临时在同一 Wi-Fi 下使用：
   在电脑这个目录运行 `python -m http.server 8787 --bind 0.0.0.0`，用 `ipconfig` 找电脑 IPv4 地址，然后在 iPhone Safari 打开 `http://电脑IP:8787/index.html`。

2. 添加到主屏幕：
   iPhone Safari 打开页面后，点分享按钮，选择“添加到主屏幕”。本地 Wi-Fi HTTP 可以作为快捷方式使用；如果想要更完整的离线缓存，建议把文件放到 HTTPS 页面上，例如 GitHub Pages。

3. 真正离线/PWA：
   项目已经包含 `manifest.webmanifest`、`service-worker.js` 和 `apple-touch-icon.png`。在 HTTPS 环境打开后，Safari 添加到主屏幕会更接近 App，并缓存核心文件。Service Worker 对页面和脚本采用 network-first：联网时总会拿到你最新发布的版本，断网时回退到缓存，所以更新代码后手机端不会一直卡在旧版本。

4. 推荐长期方案：GitHub Pages
   把这些文件推到 GitHub 仓库，并开启 GitHub Pages。之后 iPhone 直接打开 `https://你的用户名.github.io/仓库名/`，在电车上也能使用，不需要局域网。

   基本流程：
   - 新建一个 GitHub 仓库，例如 `learning-trainer`。
   - 把 `index.html`、`app.js`、`styles.css`、`manifest.webmanifest`、`service-worker.js`、`icon.svg` 推上去。
   - 在仓库 `Settings → Pages` 中选择 `Deploy from a branch`，分支选 `main`，目录选 `/root`。
   - 等 GitHub Pages 给出网址后，用 iPhone Safari 打开并“添加到主屏幕”。

## 手机-电脑同步

手机和电脑的浏览器各自保存一份 `localStorage`，不会自动互通。每天可以这样同步：

1. 在电脑或手机点“数据同步 → 生成备份”。
2. 点“复制”或“下载”，用 AirDrop、备忘录、微信文件、iCloud Drive 等方式传到另一台设备。
3. 在另一台设备打开页面，把备份文本粘贴到“备份文本”，点“导入并合并”。

导入是合并式的：自定义题、每日记录、历史和进度会尽量保留两边较新的版本。

### GitHub 自动同步

现在也支持 GitHub Gist 自动同步，适合电车上使用。

思路是：

- GitHub Pages 放网页。
- 私有 GitHub Gist 放学习进度 JSON。
- 手机和电脑打开同一个网页后，联网时自动“拉取合并 → 上传”。

设置步骤：

1. 在 GitHub 创建一个 token：
   `Settings → Developer settings → Personal access tokens`。

2. 如果用 classic token，只勾选 `gist` 权限即可。

3. 在网页的“数据同步 → GitHub 云同步”里粘贴 token。

4. 第一次使用点“创建云端”，程序会创建一个私有 Gist 并保存 Gist ID。

5. 之后保持“联网时自动同步”开启。手机上答题、保存今日记录后会自动上传；电脑打开页面后会自动拉取。

安全边界：

- Token 只存在当前浏览器的 `localStorage`，不会写进“生成备份”的 JSON。
- 不要把 token 发给别人，也不要把带 token 的浏览器配置截图公开。
- 如果你担心 token 暴露，可以只用手动“生成备份/导入并合并”。

## AI 接入说明（DeepSeek 出题）

本地规则推荐不需要联网，也不会上传你的学习记录。

如果想让程序"根据今日记录自动出题"，项目自带一个**本地 DeepSeek 代理**（`proxy/deepseek-proxy.mjs`），保证 API key 只留在你电脑上，不进前端、不进仓库。

简要用法（详见 [`proxy/README.md`](proxy/README.md)）：

```bash
# 1) 在电脑上启动代理（key 只通过环境变量传入，需 Node 18+）
DEEPSEEK_API_KEY=你的key node proxy/deepseek-proxy.mjs   # bash
# 或 PowerShell： $env:DEEPSEEK_API_KEY="你的key"; node proxy/deepseek-proxy.mjs

# 2) 网页里先保存「今日记录」，再到「AI 出题（本地代理）」面板点生成
```

设计思路：在**电脑上**生成题，生成的题进入题库并随 GitHub Gist 同步到手机，所以通勤时手机直接练，电车上和手机都不需要接触 API key。注意发布版是 HTTPS 页面，部分浏览器会拦截它访问本地 http 代理；如遇到这种情况，用 `python -m http.server 8787` 起本地网页再生成即可。

- 单元测试：运行 `node test.js`，覆盖归一化、洗牌、关键词抽取、句子切分以及数据合并等同步关键逻辑。
